# 📘 Assignment: Python Testing and Debugging

## 🎯 Objective

Learn how to verify Python code with tests and find bugs by running test cases and handling invalid input.

## 📝 Tasks

### 🛠️ Write tests for existing functions

#### Description
Use the provided starter code to write tests that verify the behavior of each function.

#### Requirements
Completed project should:

- Create test cases for `add`, `divide_numbers`, `parse_int`, `calculate_average`, and `unique_word_count`.
- Verify correct results for normal inputs.
- Include at least one test for invalid or edge-case input.
- Use a testing framework like `pytest` or a simple test runner.

### 🛠️ Debug failing code

#### Description
Run the tests, locate the functions that fail, and fix their behavior.

#### Requirements
Completed project should:

- Run the test suite and identify failing tests.
- Fix any incorrect or unexpected function behavior.
- Ensure all tests pass after your changes.
- Describe one bug you found and how you fixed it.

### 🛠️ Add error handling for invalid input

#### Description
Improve the starter code by handling invalid inputs in a safe way.

#### Requirements
Completed project should:

- Update `parse_int` to handle values that cannot be converted to integers.
- Update `calculate_average` to handle empty lists with a clear error or default.
- Add tests for these error conditions.
- Confirm the output or exception is meaningful.
